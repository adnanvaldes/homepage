Learning a Language

Five different things

  1. Syntax: how to write language constructs?
  2. Semantics: What to programs mean? How do the typechecking and evaluation rules work?
  3. Idioms: What are typical patterns for using language features to express your computation?
    1. It's not enough to iknow that if/else expressoins work; we need to know when to use them to represent and recognize patterns of use.
  4. Libraries: What facilities does the language (or a well-known project) provide as "standard"? (eg. file access, data structures, etc.)
  5. Tools: What do language implementations provide to make your job easier? (eg. REPL, debugger, code formatter, etc)
